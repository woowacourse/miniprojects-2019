소개

이 문서는 Turkeybook 서비스의 API 명세를 작성한 문서입니다.

아래 예시에서 나온 포트는 WebTestClient와 Spring MVC 간 차이 문제로 임시로 배정한 포트입니다.

요청 포트보다 `path`와 요청, 응답 형식에 집중해주세요 ^^

API에 공통으로 적용되는 사항입니다.

Cookie

TurkeyBook의 API 서버 인증은 쿠키 & 세션으로 관리하고 있습니다. 따라서 회원가입, 로그인을 제외한 모든 API 요청은 아래 쿠키를 필요로 합니다.

field

description

JSESSIONID

서버의 세션에 접근하기 위한 key

유저 생성 API 관련 형식 및 예시

Request 형식:

Path Type Description

email

String

회원의 이메일

name

String

회원의 이름

password

String

회원의 비밀번호

Response Header 형식:

Name Description

Location

생성된 유저의 정보 접근 URI

Response Body 형식:

Path Type Description

id

Number

유저의 고유 식별자

email

String

유저의 이메일

name

String

유저의 이름

login

Boolean

현재 유저의 로그인 여부

profile

Object

회원의 프로필 정보

CURL:

$ curl 'http://localhost:6849/api/users' -i -X POST \
    -H 'Accept: application/json;charset=UTF-8' \
    -H 'Content-Type: application/json;charset=UTF-8' \
    -d '{
  "email" : "UserApiCreate@gmail.com",
  "name" : "name",
  "password" : "P@ssw0rd"
}'

요청 예시:

POST /api/users HTTP/1.1
Accept: application/json;charset=UTF-8
Host: localhost:6849
Content-Length: 91
Content-Type: application/json;charset=UTF-8

{
  "email" : "UserApiCreate@gmail.com",
  "name" : "name",
  "password" : "P@ssw0rd"
}

응답 예시:

HTTP/1.1 201 Created
Date: Tue, 03 Sep 2019 05:51:39 GMT
Content-Length: 118
Content-Type: application/json;charset=UTF-8
Location: http://localhost:6849/api/users/20
Transfer-Encoding: chunked

{
  "id" : 20,
  "email" : "UserApiCreate@gmail.com",
  "name" : "name",
  "profile" : null,
  "login" : false
}

Error

User 생성과 관련된 오류 발생 조건은 다음과 같습니다.

  • password : 이메일 형식이 아닌 경우

  • name : 문자 2~10글자 형식이 아닌 경우

  • passowrd : 영대소문자, 숫자, 특수문자를 모두 포함한 8~20자

  • 이메일 중복

이 모든 오류는 400 응답코드를 반환하며 다음과 같은 필드 형식을 같습니다.

Response Body 형식:

Path Type Description

message

String
